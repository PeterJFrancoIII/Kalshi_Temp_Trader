# Recommendation and EV Layer Documentation

## Overview
The Recommendation Layer is responsible for comparing the model's output probabilities with live Kalshi market prices. It calculates the Expected Value (EV) and applies a series of safety gates to determine if a contract is a trade candidate.

## Mathematical Formulas

### 1. Market Implied Probability
We use the `yes_ask` price from the order book to determine the market's implied probability.
```
implied_probability = yes_ask / 100.0
```

### 2. Raw Edge
The difference between our model's probability and the market's implied probability.
```
edge_before_fees = model_probability - implied_probability
```

### 3. Kalshi Taker Fee
We implement the standard Kalshi taker fee formula for weather markets:
```
fee = 0.07 * price * (1.0 - price)
```
*Note: Fees are calculated in probability terms (0.0 to 1.0).*

### 4. Edge After Fees
The final statistical edge after accounting for transaction costs.
```
edge_after_fees = edge_before_fees - fee
```

## Safety Gates

Every potential trade must pass the following deterministic gates:

| Gate | Description | Threshold |
| :--- | :--- | :--- |
| **Data Staleness** | Rejects if prediction or market data is too old. | 5 minutes (300s) |
| **Market Spread** | Rejects if the bid-ask spread is too wide. | 10 cents |
| **Liquidity** | Rejects if there isn't enough depth in the order book. | 10 contracts |
| **Confidence** | Rejects if the model reports "low" confidence. | Strictly Reject "low" |
| **Mapping** | Rejects if the market contract cannot be mapped to a bin. | Strict Mapping |
| **Edge Threshold** | Promotes to `TRADE_CANDIDATE` only if edge > threshold. | 5% (0.05) |

## Recommendation Actions

- **TRADE_CANDIDATE**: Passes all gates and meets the 5% edge threshold.
- **WATCH**: Passes all safety gates, has a positive edge, but is below the 5% threshold.
- **REJECT**: Fails any safety gate or has zero/negative edge after fees.

## Audit Logs
Every recommendation generated by this module includes a detailed `reason` field, ensuring that every decision is auditable and transparent.
