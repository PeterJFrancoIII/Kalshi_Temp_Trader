# Aggregate Calibration Reporting

This document describes the aggregate calibration reporting system used to compare the performance of Model v1 (baseline rules) and Model v2 (climatology-enhanced rules) over multiple settled days.

## Purpose

The aggregate report provides a high-level statistical summary of how well the models are performing relative to each other. This is a critical step in the model development lifecycle before any real capital is deployed.

## Metrics Explained

### Accuracy Metrics
- **Average Brier Score**: The mean multi-class Brier score across all sample days. (Lower is better).
- **Average Log Loss**: The mean multi-class log loss across all sample days. (Lower is better).
- **Top-Bin Hit Rate**: The percentage of days where the bin assigned the highest probability was the actual winning bin.

### Comparison Metrics
- **Win Rate by Brier**: Percentage of days Model v2 had a lower Brier score than Model v1.
- **Win Rate by Log Loss**: Percentage of days Model v2 had a lower log loss than Model v1.
- **Win Rate by Top-Bin**: Percentage of days Model v2 hit the top bin when Model v1 did not.

### Probability Metrics
- **Average Actual Bin Probability**: The mean probability assigned to the winning bin. This measures how "close" the model's confidence was to the truth.

### Deltas (v2 minus v1)
- **Brier Delta Avg**: Average change in Brier score.
- **Log Loss Delta Avg**: Average change in log loss.

**Interpretation**:
- **Negative Delta**: Model v2 is performing better (lower error).
- **Positive Delta**: Model v1 is performing better (lower error).

## Why This Matters

Before transition to real trading, we require:
1. **Consistency**: Model v2 should demonstrate a stable win rate over a significant sample size (e.g., >30 days).
2. **Confidence Calibration**: The `avg_actual_bin_probability` should ideally match the historical frequency of those bins.
3. **Delta Magnitude**: Even if v2 wins, the magnitude of the improvement must justify the increased complexity of the model.

## Warning Handling

The aggregator will flag warnings if:
- No records are provided (`settled_days = 0`).
- Records are malformed or missing model-specific details.
- Sample size is too low for statistical significance.

## Reporting Tools

The system provides built-in writers to persist aggregate reports in multiple formats.

### JSON Writer

Saves the full structured report, including all metrics and warnings, for programmatic access.

```python
from calibration.aggregate_reports import aggregate_model_comparisons, write_aggregate_calibration_json

# Generate report
report = aggregate_model_comparisons(comparison_records)

# Save to JSON
write_aggregate_calibration_json(report, "backend/data/reports/aggregate_calibration.json")
```

### Markdown Writer

Saves a human-readable summary table and win rate list.

```python
from calibration.aggregate_reports import aggregate_model_comparisons, write_aggregate_calibration_markdown

# Save to Markdown
write_aggregate_calibration_markdown(report, "backend/data/reports/aggregate_calibration.md")
```

## Usage

### Weekly Calibration Script

The easiest way to generate an aggregate report is using the provided shell script. It automatically detects the project virtual environment and sets the necessary environment variables.

```bash
# Generate report for default KMIA production data
./scripts/generate_weekly_calibration.sh

# Generate report for a specific custom directory
./scripts/generate_weekly_calibration.sh /path/to/comparisons /path/to/output
```

### Direct CLI Module

You can also run the aggregator directly as a Python module:

```bash
python3 -m calibration.generate_aggregate_report \
    --input-dir backend/data/processed/comparisons \
    --output-dir backend/data/processed/aggregate_calibration
```

## Data Paths

- **Input**: `backend/data/processed/comparisons/*.json`
  - Each file should be a JSON comparison record as generated by the `compare-models` mode of the prediction loop.
- **Output**:
  - `backend/data/processed/aggregate_calibration/aggregate_calibration.json`: Machine-readable aggregate statistics.
  - `backend/data/processed/aggregate_calibration/aggregate_calibration.md`: Human-readable summary report.
